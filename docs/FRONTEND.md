# フロントエンド設計

## 概要
Vue 3 (Composition API) + Vite で構築されています。
状態管理にはVueの `ref` / `reactive` をコンポーネント内で使用し、グローバルストア（Pinia/Vuex）は現在の規模では使用していません。

## 主要コンポーネント構成 (`client/src/components/`)

### 1. `PartsList.vue` (メイン画面)
アプリケーションの中心となる画面です。
-   **機能**:
    -   パーツ一覧の表示（リスト表示 / グリッド表示 切り替え）
    -   検索・ソート・フィルタリング
    -   **インライン編集**: リスト表示時に名前、数量、タグなどを直接編集可能。
    -   ゴミ箱モードへの切り替え
    -   一括操作（一括削除、一括復元）
    -   詳細表示（画像プレビュー、PDF閲覧）

### 2. `PartForm.vue` (登録・編集モーダル)
パーツの新規登録および編集を行うモーダルです。
-   **機能**:
    -   マスタデータ（カテゴリ、保管場所）のプルダウン選択
    -   `TagInput` コンポーネントによるタグ入力
    -   画像・PDFのドラッグ＆ドロップまたはファイル選択
    -   カメラ起動（スマホ対応）: `<input capture>` 属性は削除済み（ライブラリ選択可にするため）

### 3. `TagInput.vue` (再利用コンポーネント)
タグ入力用の汎用コンポーネントです。
-   **機能**:
    -   入力中のタグ候補サジェスト
    -   Enterキーまたはカンマでタグ確定
    -   バックスペースでタグ削除

### 4. `MasterDataManagement.vue` (マスタ管理モーダル)
カテゴリと保管場所を管理します。
-   **機能**:
    -   新規追加
    -   **インライン名称変更**: 追加済みの項目の鉛筆アイコンから直接名称変更が可能。
    -   削除（使用中のマスタは削除注意 ※現在は警告のみ）

### 5. `DataManagement.vue` (データ管理モーダル)
バックアップと復元を管理します。
-   **機能**:
    -   CSVエクスポート/インポート
    -   Excelインポート
    -   **フルバックアップ (ZIP)**: DBとファイルをまとめてダウンロード/アップロード
    -   **データ全削除**: 危険な操作エリア（二重確認付き）

## スタイル・デザイン
-   `client/src/style.css` にグローバルスタイルを定義しています。
-   **Glassmorphism**: 半透明の背景やブラー効果を多用し、モダンなUIを実現しています。
-   **レスポンシブ**: モバイル端末でも操作しやすいレイアウトを採用しています。

## API連携 (`client/src/api.js`)
Axiosのインスタンスを作成し、ベースURLや共通設定を行っています。
コンポーネントからは `import api from '../api'` として使用します。
