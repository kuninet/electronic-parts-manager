# 電子パーツ管理アプリ

## 概要
個人の電子パーツ在庫を効率的に管理するためのWebアプリケーションです。
「パーツを買ったのにどこにあるかわからない」「在庫があるのにまた買ってしまった」といった問題を解決します。
直感的なUIで、パーツの画像、保管場所、データシートなどを一元管理できます。

## 主な機能
- **パーツ在庫管理**: 名前、説明、個数、カテゴリ、保管場所の登録・編集・削除
- **画像・資料管理**: パーツの写真やデータシート(PDF)のアップロード、URLリンク
- **検索・フィルタ**: キーワード検索、カテゴリ別・保管場所別での絞り込み表示
- **マスタ管理**: カテゴリ（例: 抵抗, IC）や保管場所（例: 工具箱A, 棚B）の編集
- **データ運用**:
  - CSV形式でのフルバックアップとリストア
  - Excelファイルからのデータインポート機能

## 技術スタック
- **Frontend**: Vue.js 3, Vite
- **Backend**: Node.js, Express, SQLite3
- **Design**: CSS (Glassmorphismを採用したモダンなダークテーマ)

## セットアップ手順

### 前提条件
- Node.js (推奨 v16以上)
- npm

### インストール方法
1. 各ディレクトリの依存関係をインストールします。

```bash
# サーバーサイド (Backend)
cd server
npm install

# クライアントサイド (Frontend)
cd ../client
npm install
```

## 起動方法

### 1. サーバーの起動
バックエンドAPIサーバーを起動します。

```bash
cd server
npm start
```
サーバーは `http://localhost:3001` で起動します。
初回起動時に `database.sqlite` が自動生成されます。

### 2. クライアントの起動
別のターミナルを開き、フロントエンド開発サーバーを起動します。

```bash
cd client
npm run dev
```
通常は `http://localhost:5173` で起動します。ブラウザでアクセスしてください。

## ディレクトリ構成

```
.
├── client/              # Vue.js フロントエンドソース
│   ├── src/             # Vueコンポーネント、アセット
│   └── index.html       # エントリーポイント
├── server/              # Node.js/Express バックエンドソース
│   ├── routes/          # APIルート定義
│   ├── database.js      # SQLite接続設定
│   ├── index.js         # サーバーエントリーポイント
│   └── database.sqlite  # データベースファイル (自動生成)
├── uploads/             # アップロードされた画像やPDFの保存先
└── README.md            # このファイル
```

## 注意事項
- `uploads` ディレクトリは画像やデータシートの保存に使用されます。削除しないよう注意してください。
- データベースファイル `server/database.sqlite` を削除するとデータが消失します。定期的なCSVバックアップを推奨します。

## QRコード入出庫管理

QRコードを使って箱（収納ケース）と部品の入出庫を管理する機能です。スマホでのQRコードスキャンに最適化されたモバイルファーストのUIを提供します。

### 機能
- **箱登録(QR)**: QRコードをスキャンして箱を登録・確認
- **部品登録(QR)**: QRコードをスキャンして部品を登録・確認
- **箱にしまう(QR)**: 箱のQR → 部品のQRをスキャンして紐付け
- **QRコード印刷**: A4全面シールに44枚のQRコードを印刷するPDF生成
- **既存データQR後付け**: パーツ編集画面からQRコードを後付け登録

### アクセス方法
メイン画面のナビゲーションから「📦 QR入出庫」リンクをクリック、またはURLに `#/qr` を付加してアクセスできます。
