【絶対遵守の厳格なワークフロールール】
※このルールはプロジェクト全体の最優先事項(GEMINI.md記載)であり、いかなる場合も例外を認めない。

1. **メインブランチ（main）への直接のソース修正・コミット・プッシュは「絶対禁止」**。
2. どんな些細な修正（1行の変更など）であっても、必ず以下の手順を厳格に守ること。
  ① まずはユーザーに問題の原因と修正方針を説明し、**「ソースを修正してよいか」の許可を求める**。
  ② 許可が出たら、**GitHub Issue を作成**する。
  ③ **専用の作業ブランチを作成**（`git checkout -b feature/...` または `hotfix/...`）し、そこで修正・コミット・プッシュを行う。
  ④ 修正後、勝手にPRを作らず、ユーザーに**「手元でのテスト・動作確認」を依頼して待機**する。
  ⑤ ユーザーから「テストOK」「マージしてよい」という**明確な許可が出た場合のみ**、Pull Request を作成し、main にマージし、Issue をクローズする。
