# タスク完了チェックリスト (Task Completion Checklist)

タスク完了時に以下の手順を確認すること:

1.  **検証 (Verify)**: 機能が期待通り動作することを確認する（手動またはスクリプト）。
2.  **Lint**: コードが綺麗で読みやすい状態か確認する。
3.  **ドキュメント (Docs)**: アーキテクチャやAPIに変更があった場合は `docs/` を更新する。
4.  **成果物 (Artifacts)**: 機能追加や修正があった場合は `walkthrough.md` を更新する。
5.  **通知 (Notify)**: 変更内容のサマリをユーザーに報告する。
